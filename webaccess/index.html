In this project we had to evaluate a robust, multi-page website on the basis of accessibility. We had to identify 3 accessibility problems using a web accessibility tool. For my project, I used the WAVE tool. With that, we had to list the WCAG 2.1 Success Criterion that was violated, with a brief description. After that,we had to come up with recommended accessibility improvements, whether on the basis of code, design or content. We used the WAI template to report the suggestions.

Website Information

Name of website: LingCars
Purpose of site: To provide car leasing services in the UK.
Base URL of site: https://www.lingscars.com/ (Links to an external site.)

Problem #1: The contrast between the foreground and background colors is very low.

Problem Description: In some text and images, the color of the text doesn’t contrast enough, which makes it hard to read for people with low vision. Text is present that has a contrast ratio less than 4.5:1, or large text (larger than 18 point or 14 point bold) has a contrast ratio less than 3:1. It’s also relevant for text with background images, where the color of the background should have enough contrast with the text.
Problem Impact: Users with visual impairment would have difficulty reading text if it’s color is too close to the background. Readers often use contrast as a way to interpret the letters, with its shape visible

Relevant WCAG 2.1 Guidelines and Success Criteria:

1.4.3 Contrast (Minimum) (Level AA): Text and images of text have a contrast ratio of at least 4.5:1.

Solution

To increase the contrast between text and background, through making the text color black(#000000) or a dark grey of #9E9E9E.

<p style="color: #9e9e9e;">terms and conditions</p> <p style="color: black;">Terms and conditions</p>

Problem #2: Image doesn't have alternative text on website

Problem Description: In order to get pictures across to the screen reader, it should have alternative text. Some images on the screen do not have alternative text, and so if the user were to use screen reader, they wouldn’t be able to understand the text because it doesn’t have a description.
Problem Impact: Users with vision impairment who use screen readers would find it difficult to understand what the image is without alternative text in it. If the image isn’t available, it would be hard to interpret it. Thus, all images should have an alternate text attribute.

Relevant WCAG 2.1 Guidelines and Success Criteria:

1.1.1 Non-text Content (Level A): Images, form image buttons, and image map hotspots have appropriate, equivalent alternative text.

Solution

While inserting an image, insert the alternate attribute to the tag

<img src="img/nissan leaf.JPG.JPG" alt="Nissan Leaf"> <img src="img/checker flag.JPG" alt="Checker Flag"> <img src="img/lightning bolt.JPG" alt="lightning bolt">

problem description: Since adjecent links go to the same URL, it can re redundant for the user, especially someone who is using assistive technology like a screen reader. Problem Impact: Users who use assistive technologies like screen readers may feel confused.

Problem #4: One of the Elements of the website has a "Marquee" tool.

Problem Description: There is a Marquee element present on the website, which presents text that scrolls across the page, out of the users control. Users cannot stop to read or evaluate the text. It could also be a distracting factor to users, especially those with some cognitive disabilities. The text scrolling is too fast, and with a lot of text that makes it hard to follow.
Problem Impact: Users with cognitive disabilities would find it really hard to read and follow this text. It could be very distracting to users, since it's always in motion. Since users are unable to stop the text, they have to read quickly, which can be a hindrance to some.

Relevant WCAG 2.1 Guidelines and Success Criteria:

2.2.6 Timeouts(WCAG 2.1 Level AAA): Users must be warned of any timeout that could result in data loss, unless the data is preserved for longer than 20 hours of user inactivity.
2.2.2 Pause, Stop, Hide (Level A) : Automatically moving, blinking, or scrolling content (such as carousels, marquees, or animations) that lasts longer than 5 seconds can be paused, stopped, or hidden by the user.

Solution

Replace marquee tool with standard HTML element and style it with CSS.

<p>Leader of the Pack - The UK's favorite car leasing website! </p>

Or add a stop and start button for the marquee

<marquee behavior="scroll" direction="left" scrollamount="10" id="marquee2"><p>Leader of the Pack - The UK's favorite car leasing website!</p></marquee> <input type="button" value="Stop Marquee 1" onClick="document.getElementById('marquee2').stop();"/> <input type="button" value="Start Marquee 1" onClick="document.getElementById('marquee2').start();"/><br />
